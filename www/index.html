SubEtha is a sophisticated mailing list manager, similar in many respects to the popular Mailman package. SubEtha's features include:

  • Very easy installation on Windows and Unix platforms
  • A user-friendly web interface for all configuration management
  • Virtual domains (ie list@foo.com and list@bar.com are separate lists)
  • Intelligent VERP bounce processing
  • Message processing 'mailets' which allow administrators to write plugins which can modify the message stream to allow features we plan to offer, such as stripping of MIME attachments, spam detection and insertion of advertising.
  • Users can have multiple email addresses on a single list delivered to a preferred address
  • Users can self-moderate messages sent from unsubscribed addresses
  • Searchable, threaded archives implemented with a UI that is similar to forum software with access control based viewing and posting
  • Clusterable for nearly unlimited scalability
  • Easy integration with any mail transport agent (MTA)
  • EJB and SOAP interfaces for automation

SubEtha is a three-tiered J2EE application using EJB3 and JMS. It is designed to deploy on JBoss and relies on numerous other open source projects including Hibernate, Tagonist, and Lucene.

This project is still in the design phase. Its authors are:

  • Jeff Schnitzer
  • Scott Hernandez
  • Ian McFarland
  • Jon Stevens

There will be two distributions. The first, for advanced administrators, is an EAR file which can be deployed into any JBoss instance. The second, for novices, is a "just-unzip-and-run" package containing standalone JBoss with SubEtha preconfgured and all unnecessary services removed.
